Problema 1 (Para todos os alunos)

Neste problema você deverá utilizar d3 para desenvolver uma função que desenha um gráfico de pizza (ou pie chart). A solução deste problema deverá conter dois arquivos problema1.html e problema1.js. No arquivo .js, deverá ser implementada uma função chamada pieChart com a seguinte assinatura¨function pieChart(probabilities,colors), onde probabilities é um array contendo as probabilidades (porção da área de cada fatia) e colors é um array de mesmo comprimento que probabilities contento as cores que deverão ser atribuídas às fatias. Para a solução deste problema não será permitido o uso do pie layout do D3, isto é, as soluções deveram ser implementadas utilizando elementos básicos do SVG e D3.

Problema 2 (Para todos os alunos)

Durante a última aula prática, vimos como usar transições em D3. Neste problema você irá utilizar o código desenvolvido no problema 1 para ter um pie chart animado. Para tando use o esqueleto de código presente nos arquivos lista2_problema2.html,lista2_problema2.js e colorbrewer.min.js. Você deverá modificar estes arquivos e entregar a sua versão final como solução. Um exmplo de como a animação deve ser comportar, pode ser visto aqui (node que o seu pie chart não precisa necessariamente ser igual ao mostrado nesse link, use o link para ver o comportamento da animação).

Problema 3 (Para todos os alunos)

Abra a página lista2_problema3.html. Uau! Nosso velho scatterplot! Temos código para gerar uma visualização para um conjunto de dados geral!
  1. O que faremos agora é incluir nesse html o script dataset.js, que contém um dataset um pouco mais interessante de ser visualizado. Depois de ter feito isso, remova a definição do dataset no arquivo lista2_problema3.js. Depois de ter feito isso, você provavelmente verá o resultado incrível do scatterplot nesse novo dataset! O que você aprendeu a partir dessa visualização? (provavelmente não muito). Por que isso ocorreu? O que você faria para modificar isso? Sugira possíveis soluções para o problema e tente implementá-las. Escreva suas considerações num arquivo texto, chamado problema3_1.txt.
  2. Agora começaremos a adicionar funcionalidades de interação do usuário em nossas visualizações. Para tanto, incluiremos a capacidade de selecionar pontos no nosso scatterplot. Desejamos ser capazes de usar o mouse para desenhar um retangulo que seja capaz de selecionar pontos. Use como base este código. Modifique a função que responde ao evento de "mouseup" para que quando o botão do mouse for solto, os pontos dentro do retângulo sejam coloridos de vermelho. Novamente, você não deve usar nenhuma função pré-definida em D3 para fazer seleções. Como dica, você provavelmente vai querer dar uma olhada na função invert das escalas em D3.

Problema 4 (Obrigatório apenas para alunos de pós-graduação)

Modifique o códico do problema 3 para:
  1. Suportar seleções "persistentes", ou seja, os retângulos da seleção não devem "apagar" ao botão do mouse ser solto.
  2. Seu código deve ser capaz de suportar duas seleções simultâneas, identificadas como azul e laranja. Os pontos selecionados (dentro de cada retângulo) devem ser coloridos de acordo com a cor da seleção.
  3. Deverá ser possível de clicar em um retângulo de seleção e arrastá-lo (dragging) pela tela, e a cor dos pontos selecionados deve ser alterada de acordo.
  4. Para cada uma das seleções criadas, a interface deve ser capaz de plotar a regressão linear dos pontos selecionados por cada retângulo. A reta de regressão deve ser plotada com a cor da seleção correspondente.
